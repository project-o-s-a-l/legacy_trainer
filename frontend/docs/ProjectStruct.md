# LegacyTrainer - структура директорий frontend
### Глобальные вещи (Файлы/директории):
То, что влиает а все приложение, не должно лежать внутри страницы или виджета
К примеру:
- Глобальные CSS-стили / переменные;
- Theme provider;
- API Client;
Так же есть файлы index.ts - это файлы-импорты. Они существуют почти в каждой дириктории. Нужны для удобных импортов.
К примеру:
```TypeScript
export { default } from "./Navbar";
```
Использование:
```TypeScript
export Navbar from "../widgets/Navbar";
```
В index.ts может быть так же несколько импоротов:
```TypeScript
export { default as LoginForm } from "./ui/LoginForm";
export { default as SingOutButton } from "./ui/SignOutButton";
export { useAuth } from "./model/useAuth";
```
Использование:
```TypeScript
import { LoginForm, useAuth } from "../features/auth";
```

## src/
Основная директория, находятся директории с tsx-файлами, css-файлами, ts-файлами.
Обязательно здесь находится точка входа в приложение (index.tsx). Служебные TypeScript/CRA файлы, основные слои архитектуры
#### Подробно:
##### index:
index.tsx - точка входа для react-scripts
Что обычно происходит:
- Импортируется глобальный CSS;
- Подключается App;
- Рендерится React root;
##### react-app-env.d.ts:
- Служебный файл для TypeScript в CRA;
##### reportWebVitals.ts:
- Файл для метрик производительности;
##### setupTests.ts:
Общая настройка тестового окружения;
---
## app/
Слой приложения целиком. Кладем, то что относится ко всему сайту сразу
#### Подрбно:
##### App.tsx:
Корневой компонент приложения. Внутри обычно:
- layout-обёртка;
- провайдеры;
- router rendering;
- глобальные boundary-компоненты;
##### providers/:
Глобальные провадйеры
###### providers/router/:
Все что связано с маршрутизацией:
- renderRoutes.tsx - генерация <Route /> 
- routeConfig.ts - список маршрутов
##### styles/
Глобальные стили проекта
###### styles/*.css
- index.css: Главный CSS-файл, импоритрует остальные глобальные стили
- variables.css: 
    - цвета;
    - отступы;
    - ширина контейнера;
    - размеры шрифты;
    - радиусы;
- typography.css: общие стили типографики
##### config/
- env.ts - чтение и нормализация env-переменных
- constatns.ts - константы верхнего уровня
---
## pages/
Каждая папка - отдельная страница маршрута
Класть только когда, это отдлеьная страница маргрута
Обычно состоит из:
- widgets ( header, footer, navbar);
- Подключает features;
- формирует layout;
- не должна быть свалкой всей логики приложения;
---
## widgets/
Это крупные готовые блоки интерфейса
Обычно:
- Navbar;
- Footer;
- Sidebar;
---
## features/
Пользовательские сценарии и бизнес логика.
Класть всё что относится к действием пользователя.
- login;
- search;
- switch theme;
- fitlers;
### Внутреняя структура:
#### ui/
- Формы;
- Кнопки действий;
- модалки;
#### model/
Внутреняя логика фичи:
- типы;
- хуки;
- slice;
- selectors;
- state;
#### api/
API-запросы фичи
## shared/
Переиспользуемый фундамент проекта.
Класть всё что не принадлежит к одной конкретной странице/фиче;
### assets/
Глобальные ассеты:
- images/ - изображения;
- logo/ - логитип;
- fonts/ - шрифты;
### ui/
Базовые UI-компоненты. 
Это универсальные кирпичики интерфейса.
Примеры:
- Button;
- Input;
- Modal;
### types/
Обшие TypeScript-типы
Примеры: 
- navbarProps.ts;
- routes.ts;
### config/
Это настройка проекта (Пока эта часть под вопросом)
Класть если это настройки, пути, ключи, конфиги, которые используются в проекте повторно.
### constants/
Это общие константы (Пока тоже под вопросом)
Пример: в App.ts содержится export const APP_NAME = "Name_Project";

# Краткий "гайд" по организации:
**Это относится ко всему приложению?**
Тогда app/
**Это отдельная страница?**
Тогда pages/
**Это большой визиуальный блок?**
Тогда widgets/
**Это действие пользователя?**
Тогда features/
**Это что-то общее для разных мест?**
тогда shared/
